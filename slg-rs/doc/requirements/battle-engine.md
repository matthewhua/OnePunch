# 战斗引擎需求文档

> 对应 Phase：Phase 10（战斗部分）
> 优先级：P2
> 预估工期：3-4 周

---

## 一、概述

战斗引擎是 SLG 游戏的核心计算模块，负责回合制战斗的数值计算、战报生成和结果应用。对应 Java 版的 `FightService` + `BattleReport` 体系。

---

## 二、功能需求

### 2.1 战斗模型

#### FR-BT-001：回合制战斗
- 战斗分为多个回合（最多 30 回合）
- 每回合：攻方行动 → 防方行动
- 行动顺序由将领速度属性决定
- 回合结束检查胜负条件

#### FR-BT-002：兵种系统
| 兵种 | 克制关系 | 特点 |
|------|---------|------|
| 步兵 | 克骑兵 | 高防御 |
| 骑兵 | 克弓兵 | 高速度 |
| 弓兵 | 克步兵 | 高攻击 |
| 攻城车 | 对建筑加成 | 低速度 |

#### FR-BT-003：伤害计算
```
基础伤害 = 攻击方攻击力 × 兵力系数 - 防御方防御力 × 防御系数
克制加成 = 基础伤害 × 克制系数（1.3 克制 / 0.7 被克制 / 1.0 无克制）
技能加成 = 将领技能效果
最终伤害 = (基础伤害 + 克制加成) × (1 + 技能加成) × 随机波动(0.95~1.05)
```

#### FR-BT-004：将领技能
- 主动技能：每 N 回合触发一次
- 被动技能：满足条件自动触发
- 技能效果：伤害、治疗、增益、减益
- 技能目标：单体、群体、自身

### 2.2 战斗流程

#### FR-BT-005：战斗初始化
1. 收集攻方数据：将领属性、兵力、科技加成、装备加成
2. 收集防方数据：将领属性、兵力、城防加成（攻城战）
3. 计算最终属性（基础 + 所有加成）
4. 初始化战斗状态

#### FR-BT-006：回合执行
1. 确定行动顺序（按速度排序）
2. 每个将领依次行动：
   a. 检查技能是否可释放
   b. 选择目标
   c. 计算伤害/效果
   d. 应用伤害（减少兵力）
   e. 检查是否有单位被消灭
3. 回合结束，检查胜负

#### FR-BT-007：胜负判定
- 一方兵力全部归零 → 另一方胜利
- 达到最大回合数 → 防守方胜利（攻城战）/ 兵力多的一方胜利（野外）
- 平局处理

### 2.3 战报系统

#### FR-BT-008：战报生成
- 记录每回合的详细行动（谁攻击谁、伤害多少、技能触发）
- 记录最终结果（胜负、双方损失、掠夺资源）
- 战报序列化为 protobuf 格式
- 战报可在客户端回放

#### FR-BT-009：战报存储
- 战报通过邮件发送给双方玩家
- 战报保留 7 天
- 重要战报（攻城战、联盟战）保留 30 天

### 2.4 战斗结果应用

#### FR-BT-010：兵力损失
- 攻击方：损失的兵力按比例分为死亡和伤兵
- 防御方：损失的兵力按比例分为死亡和伤兵
- 伤兵可在医院恢复
- 死亡兵力永久损失

#### FR-BT-011：资源掠夺
- 攻城战胜利：掠夺防守方仓库中未保护的资源
- 掠夺量受攻击方负重限制
- 掠夺的资源在部队返回后入账

#### FR-BT-012：经验和积分
- 战斗胜利获得将领经验
- 战斗结果影响排行榜积分
- 战斗结果触发任务/活动进度

## 三、技术设计

### 3.1 战斗引擎独立性
- 战斗引擎是纯计算模块，不依赖网络和数据库
- 输入：攻防双方的属性数据
- 输出：战斗结果和战报
- 可独立进行单元测试

### 3.2 代码结构
```
crates/shared/src/battle/
├── mod.rs           # 战斗引擎入口
├── model.rs         # 战斗数据模型
├── calculator.rs    # 伤害计算
├── skill.rs         # 技能系统
├── round.rs         # 回合执行
├── report.rs        # 战报生成
└── result.rs        # 结果应用
```

### 3.3 性能要求
- 单场战斗计算 < 1ms
- 支持批量战斗（集结战、联盟战）
- 战斗计算不阻塞 Actor 消息处理

## 四、与 Java 版的兼容

### 4.1 短期方案
- 复用 Java 版战斗引擎（通过 gRPC 调用 Java fight 模块）
- Rust 侧只负责触发和结果应用

### 4.2 长期方案
- 在 Rust 侧完整实现战斗引擎
- 确保计算结果与 Java 版一致（数值验证）
- 逐步替换 Java 版
