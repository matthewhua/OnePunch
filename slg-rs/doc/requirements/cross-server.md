# 跨服系统需求文档

> 对应 Phase：Phase 10（跨服部分）
> 优先级：P3
> 预估工期：3-4 周

---

## 一、概述

跨服系统允许不同服务器的玩家进行交互和竞技。对应 Java 版的 `CrossService` + `imperial_sim_component/cross` 模块。

---

## 二、功能需求

### 2.1 跨服基础设施

#### FR-CS-001：跨服通信
- 各游戏服通过 gRPC 与跨服服务通信
- 跨服服务作为中心节点协调多个游戏服
- 通信协议：gRPC + protobuf
- 支持消息可靠投递（重试 + 确认）

#### FR-CS-002：跨服匹配
- 根据服务器等级/战力进行分组匹配
- 匹配算法：按服务器开服时间和平均战力分组
- 匹配结果缓存，避免频繁重新匹配

#### FR-CS-003：跨服数据同步
- 玩家参与跨服活动时，将必要数据同步到跨服服务
- 同步数据：玩家基础信息、将领数据、部队数据
- 跨服活动结束后，将结果同步回各游戏服

### 2.2 跨服玩法

#### FR-CS-004：跨服竞技场
- 不同服务器的玩家进行 1v1 对战
- 匹配机制：按战力段位匹配
- 赛季制：每周/每月重置排名
- 排名奖励

#### FR-CS-005：跨服联盟战
- 不同服务器的联盟进行对抗
- 攻城战模式：攻占对方据点
- 积分制：按战斗结果累计积分
- 赛季奖励

#### FR-CS-006：跨服世界 Boss
- 多个服务器的玩家共同挑战世界 Boss
- Boss 血量根据参与服务器数量调整
- 伤害排名奖励

#### FR-CS-007：跨服王城争夺
- 多个服务器争夺中央王城
- 阶段制：预选赛 → 淘汰赛 → 决赛
- 占领王城的服务器获得全服加成

### 2.3 跨服管理

#### FR-CS-008：跨服赛季管理
- 赛季开启/关闭
- 赛季结算和奖励发放
- 赛季数据归档

#### FR-CS-009：跨服服务器分组
- 服务器分组管理（哪些服务器在同一个跨服组）
- 支持动态调整分组
- 新服加入跨服组的时机控制

## 三、技术设计

### 3.1 架构
```
Game Server A ──┐
Game Server B ──┼── gRPC ──▶ Cross Service ──▶ Cross DB
Game Server C ──┘
```

### 3.2 跨服服务独立部署
- Cross Service 作为独立进程部署
- 管理跨服活动的生命周期
- 存储跨服排名和战斗数据

### 3.3 数据一致性
- 跨服操作使用分布式事务或最终一致性
- 关键操作（奖励发放）使用幂等设计
- 网络分区时的降级策略
