# World 服务需求文档

> 对应 Phase：Phase 4
> 优先级：P1
> 预估工期：4-6 周

---

## 一、概述

World Service 管理世界地图、行军、战斗等全服共享数据。采用 MapSectorActor 分区模型，每个分区管理一块地图区域。对应 Java 版的 `Game Server` 中世界地图相关逻辑（`GlobalActor` + `WorldMapService` + `MarchService`）。

---

## 二、功能需求

### 2.1 地图系统

#### FR-WD-001：地图格子管理
- 世界地图 1300×1300 格子（可配置）
- 每个格子包含：坐标、地形类型、实体信息、所属分区
- 格子实体类型：空地、玩家城池、NPC 据点、资源点、装饰物
- 支持格子查询：按坐标、按区域范围、按实体类型

#### FR-WD-002：地图分区
- 地图按区域划分为 N×N 个 Sector（默认 4×4 = 16 个分区）
- 每个 Sector 是一个独立的 MapSectorActor
- 分区大小可配置（影响单 Actor 负载和跨区频率的平衡）
- 坐标 → 分区 ID 的映射函数

#### FR-WD-003：地图初始化
- 服务启动时从 MySQL 加载地图数据
- 初始化 NPC 据点、资源点等固定实体
- 玩家城池位置从 p_global 加载

#### FR-WD-004：地图实体管理
- 实体 CRUD：创建、查询、更新、删除
- 实体类型：
  - 玩家城池（PlayerCity）：位置、等级、防御力、所属联盟
  - NPC 据点（NpcBase）：类型、等级、刷新时间、守军
  - 资源点（ResourcePoint）：资源类型、剩余量、采集者
  - 野怪（Monster）：等级、刷新时间
  - 装饰物（Decoration）：纯展示

### 2.2 AOI 系统

#### FR-WD-005：视野管理
- 每个玩家有一个视野范围（默认 9 宫格，即周围 8 个格子 + 自身）
- 玩家打开地图时订阅视野范围内的事件
- 视野移动时更新订阅（取消旧区域、订阅新区域）
- 玩家关闭地图或下线时取消所有订阅

#### FR-WD-006：AOI 事件广播
- 视野范围内发生变化时推送给订阅者：
  - 实体进入视野（EntityEnter）
  - 实体离开视野（EntityLeave）
  - 实体状态更新（EntityUpdate）：行军位置、采集进度
  - 部队出发（MarchStart）
  - 部队到达（MarchArrive）
  - 战斗发生（BattleOccurred）
- 广播通过 Gateway 推送给客户端

### 2.3 行军系统

#### FR-WD-007：派兵出征
- 玩家选择将领和兵力，指定目标坐标
- 校验：将领是否空闲、兵力是否足够、目标是否合法
- 计算行军时间（基于距离和速度加成）
- 创建 Troop 实体，加入时间轮

#### FR-WD-008：行军类型
| 类型 | 说明 | 到达后行为 |
|------|------|-----------|
| Attack | 攻击玩家城池 | 触发 PvP 战斗 |
| AttackNpc | 攻击 NPC 据点 | 触发 PvE 战斗 |
| Gather | 采集资源 | 开始采集倒计时 |
| Scout | 侦查 | 返回目标信息 |
| Rally | 集结 | 等待集结完成后统一出发 |
| Return | 返回 | 部队回城，归还将领和兵力 |
| Reinforce | 驻防 | 驻扎在目标位置 |

#### FR-WD-009：行军控制
- 召回：行军中的部队可以召回，立即掉头返回
- 加速：使用道具减少剩余行军时间
- 行军位置插值：客户端根据出发时间和到达时间计算当前位置

#### FR-WD-010：跨区行军
- 部队从 Sector A 行军到 Sector B 时，需要跨 Actor 转移
- 转移流程：Sector A 检测到部队离开 → 发送 TransferTroop → Sector B 接收
- 转移前写 WAL 日志，防止转移过程中数据丢失

### 2.4 战斗触发

#### FR-WD-011：PvP 战斗触发
- 攻击部队到达目标城池时触发
- 收集攻防双方数据（将领、兵力、科技加成、装备加成）
- 调用战斗引擎计算结果
- 发送战报给双方（通过 Home Service）

#### FR-WD-012：PvE 战斗触发
- 攻击 NPC 据点时触发
- NPC 守军数据从静态配置加载
- 战斗结果影响 NPC 据点状态（占领/击败）

### 2.5 采集系统

#### FR-WD-013：资源采集
- 部队到达资源点后开始采集
- 采集速度 = 基础速度 × 科技加成 × VIP 加成
- 采集完成或资源耗尽时部队自动返回
- 采集中被攻击时触发战斗

#### FR-WD-014：资源点管理
- 资源点定时刷新（从静态配置读取刷新规则）
- 资源点类型：粮食、木材、铁矿、石料、金币
- 资源点等级影响产出量和采集速度

### 2.6 NPC 系统

#### FR-WD-015：NPC 据点
- NPC 据点从静态配置加载
- 据点类型：野怪营地、资源矿、关卡、世界 Boss
- 据点被击败后进入冷却期，冷却结束后刷新
- 高级据点需要联盟集结攻打

#### FR-WD-016：NPC 刷新
- 定时刷新机制（每小时/每天/每周）
- 刷新时恢复守军、重置状态
- 刷新规则从静态配置读取

---

## 三、非功能需求

### 3.1 性能
- 单 Sector tick 处理时间 < 50ms（200ms tick 间隔）
- 支持 10000+ 同时行军的部队
- AOI 广播延迟 < 100ms

### 3.2 可靠性
- Sector Actor panic 后自动重启
- WAL 保证行军数据不丢失
- 跨区转移的原子性保证
