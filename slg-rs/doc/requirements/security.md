# 安全与反作弊需求文档

> 优先级：P2
> 预估工期：1-2 周

---

## 一、概述

游戏服务端需要防范协议篡改、频率攻击、数据注入等安全威胁。本文档定义了通信安全、数据校验和反作弊的需求。

---

## 二、功能需求

### 2.1 通信安全

#### FR-SE-001：TLS 加密
- 生产环境强制使用 TLS 1.3
- 证书管理：支持 Let's Encrypt 自动续期
- 开发环境可关闭 TLS

#### FR-SE-002：协议加密
- 在 TLS 基础上，对 protobuf body 进行额外加密（可选）
- 加密算法：AES-256-GCM
- 密钥协商：登录时通过 ECDH 交换密钥
- 加密开关可配置（开发环境关闭）

#### FR-SE-003：防重放攻击
- 每个请求携带递增的序列号（sequence）
- 服务端校验序列号单调递增
- 重复序列号的请求直接丢弃

### 2.2 频率限制

#### FR-SE-004：连接级限流
- 单连接每秒最大消息数：100（可配置）
- 超限后返回 ServerBusy 错误码
- 持续超限 10 秒断开连接

#### FR-SE-005：IP 级限流
- 单 IP 每分钟最大登录次数：10
- 单 IP 最大并发连接数：5
- 超限后拒绝新连接

#### FR-SE-006：命令级限流
- 特定高消耗命令有独立的频率限制
- 例如：聊天消息每 3 秒一条、商店购买每秒一次
- 限流规则可配置

### 2.3 数据校验

#### FR-SE-007：请求参数校验
- 所有客户端请求的参数进行合法性校验
- 数值范围检查（如数量不能为负数）
- 枚举值检查（如兵种类型必须合法）
- 字符串长度检查（如昵称长度限制）

#### FR-SE-008：业务逻辑校验
- 操作前置条件检查（如升级建筑需要足够资源）
- 状态一致性检查（如将领出征中不能再次出征）
- 时间合法性检查（如加速不能超过剩余时间）

#### FR-SE-009：服务端权威
- 所有数值计算在服务端完成
- 客户端只发送操作意图，不发送计算结果
- 服务端校验后返回最终结果

### 2.4 反作弊

#### FR-SE-010：异常行为检测
| 行为 | 检测方式 | 处理 |
|------|---------|------|
| 加速器 | 操作频率异常高 | 警告 → 封号 |
| 协议篡改 | 参数超出合法范围 | 断开连接 |
| 多开 | 同 IP 多个账号 | 限制 |
| 脚本挂机 | 操作模式固定 | 验证码 |

#### FR-SE-011：操作日志
- 记录所有关键操作（充值、交易、战斗）
- 日志包含：时间、玩家、操作类型、参数、结果
- 日志保留 90 天
- 支持按玩家查询操作历史

#### FR-SE-012：封禁系统
- 支持封禁账号（永久/临时）
- 支持封禁 IP
- 支持禁言（聊天限制）
- 封禁记录和解封功能

### 2.5 敏感数据保护

#### FR-SE-013：密码和 Token 安全
- 密码使用 bcrypt/argon2 哈希存储
- Session token 使用 HMAC 签名
- Token 不在日志中明文输出

#### FR-SE-014：SQL 注入防护
- 使用 sqlx 的参数化查询（已天然防护）
- 禁止拼接 SQL 字符串

## 四、配置项

```toml
[security]
# TLS
tls_enabled = false
tls_cert_path = "certs/server.crt"
tls_key_path = "certs/server.key"

# 协议加密
payload_encryption = false

# 限流
rate_limit_per_connection = 100    # 每连接每秒
rate_limit_per_ip_login = 10       # 每 IP 每分钟登录次数
max_connections_per_ip = 5

# 防重放
sequence_check_enabled = true

# 封禁
ban_check_cache_ttl_secs = 300
```
